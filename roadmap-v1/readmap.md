# AI 开发者成长计划（目标 + 实践）

## 使用说明
- 学习周期：6 个月
- 每周投入：8-12 小时
- 时间分配：30% 学习 + 60% 实践 + 10% 复盘
- 执行原则：只学当前项目需要的知识，学完立即实战，做完立即复盘

## 总体目标
- 从“会用 AI 工具”进阶到“能独立构建 AI 应用”
- 从“能跑通 Demo”进阶到“可评估、可优化、可交付”
- 建立完整能力闭环：需求 -> 实现 -> 评估 -> 优化 -> 迭代

---

## 阶段一（第 1-4 周）：会用 AI 工具解决具体问题

### 阶段目标
- 熟练调用 LLM API
- 掌握 Prompt 模板化编写
- 能完成一个可演示的小工具

### 实践项目
- 项目名：个人效率助手
- 功能建议：文本总结、改写润色、提纲生成、邮件草稿

### 每周目标 + 实践
1. 第 1 周  
   目标：跑通 API 调用和基础提示词  
   实践：实现命令行版本，支持 1-2 个任务（如总结、翻译）

2. 第 2 周  
   目标：提升输入输出稳定性  
   实践：增加结构化输出（JSON/Markdown）与异常处理

3. 第 3 周  
   目标：支持多轮与历史记录  
   实践：加入本地会话记录（文件存储）和上下文管理

4. 第 4 周  
   目标：形成可演示 MVP  
   实践：完成一个可运行版本（CLI 或 Web 均可）

### 阶段验收
- 可稳定完成至少 3 类文本任务
- 有基本日志与错误记录
- 有 README（功能、用法、已知限制）

---

## 阶段二（第 5-8 周）：掌握 RAG，做知识库问答

### 阶段目标
- 理解 RAG 核心链路：切分 -> 向量化 -> 检索 -> 生成
- 能定位回答错误来自检索还是生成

### 实践项目
- 项目名：文档问答助手
- 数据来源：个人笔记、技术文档、项目文档

### 每周目标 + 实践
1. 第 5 周  
   目标：完成知识入库流程  
   实践：实现文档切分、向量化、向量库存储

2. 第 6 周  
   目标：跑通问答链路  
   实践：实现检索 + 生成，回答附引用来源

3. 第 7 周  
   目标：提升检索质量  
   实践：调优 chunk 大小、topK、重排策略并对比结果

4. 第 8 周  
   目标：建立基础评测  
   实践：制作 20-30 题评测集，记录命中率/准确率

### 阶段验收
- 回答可追溯到文档引用
- 有至少 2 套检索参数对比结论
- 能解释主要错误来源和改进方向

---

## 阶段三（第 9-12 周）：做 Agent，完成多步骤任务

### 阶段目标
- 让模型具备任务拆解和工具调用能力
- 能完成多步骤自动化任务

### 实践项目
- 项目名：任务执行 Agent
- 场景示例：收集资料 -> 整理摘要 -> 输出报告

### 每周目标 + 实践
1. 第 9 周  
   目标：定义工具能力边界  
   实践：实现工具接口（搜索、读写、计算）

2. 第 10 周  
   目标：实现任务分解与执行  
   实践：加入步骤规划、工具调度、失败重试

3. 第 11 周  
   目标：提升可观测性  
   实践：记录每一步输入、输出、耗时、错误原因

4. 第 12 周  
   目标：评估稳定性  
   实践：回放 10 个真实任务并统计成功率

### 阶段验收
- 可独立完成 3 步以上任务
- 出错可定位到具体步骤
- 可统计时延、成功率、调用成本

---

## 阶段四（第 13-18 周）：从“能跑”到“好用”

### 阶段目标
- 建立评估体系与优化闭环
- 让系统具备可持续迭代能力

### 实践任务
- 建立离线评测集（正确性、一致性、引用质量）
- 建立在线指标（时延、错误率、反馈评分）
- 开展 A/B 对比（Prompt、模型、检索策略）
- 实施优化（缓存、上下文裁剪、模型路由）

### 阶段验收
- 每次改动都有前后对比数据
- 有固定评估报告模板
- 单次请求成本可解释并可优化

---

## 阶段五（第 19-24 周）：交付可上线的小产品

### 阶段目标
- 完成从需求到交付的完整闭环
- 支持小范围真实用户试用

### 实践项目
- 任选一个业务场景：
  - 客服助手
  - 知识运营助手
  - 代码辅助助手

### 执行步骤
1. 需求定义：目标用户、核心场景、成功指标  
2. 技术方案：模型选型、RAG/Agent、存储与权限  
3. MVP 开发：可运行、可演示、可收集反馈  
4. 小范围试用：收集问题并迭代优化  
5. 产出文档：产品说明、技术说明、复盘报告

### 阶段验收
- 至少 5 位真实用户试用
- 至少 2 轮可追踪迭代
- 支持稳定演示与部署

---

## 每周复盘模板（固定执行）
- 本周目标是否达成（是/否 + 原因）
- 本周成果（功能、截图、日志、指标）
- 关键问题 3 个（现象、原因、改进动作）
- 下周最重要 3 件事
- 一个量化指标（准确率/成功率/时延/成本）

## 每月里程碑检查
- 是否有可运行版本
- 是否有可复现指标
- 是否有明确迭代方向
- 是否形成文档沉淀（README + 复盘）

## 建议的长期主线项目
- 统一主线：AI 助手平台
- 演进路径：单模型问答 -> RAG -> Agent -> 评估系统 -> 模型路由
- 价值：所有学习内容都能持续叠加到同一项目，成长效率最高
