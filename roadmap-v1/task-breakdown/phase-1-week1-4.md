# 阶段一任务拆分（第 1-4 周）

## 目标
- 熟练调用 LLM API
- 掌握 Prompt 模板化编写
- 交付可演示的小工具（个人效率助手）

## 第 1 周：跑通基础调用
- 任务内容：完成模型 API 接入与最小命令行调用
- 怎么执行：
  - 选定模型平台并配置环境变量
  - 编写 `prompt -> model -> output` 最小脚本
  - 增加命令行参数（输入文本、任务类型）
- 产出什么：
  - 可执行脚本（CLI）
  - 使用说明（README）
  - 3 组样例结果

## 第 2 周：结构化输出与容错
- 任务内容：让输出可解析、流程可恢复
- 怎么执行：
  - 增加固定输出模板（JSON 字段约束）
  - 对超时/空结果/接口报错增加重试和降级提示
  - 落地请求日志与错误日志
- 产出什么：
  - 结构化输出版本
  - 日志文件
  - 异常测试记录（至少 5 条）

## 第 3 周：多轮对话和历史记录
- 任务内容：支持连续对话和上下文管理
- 怎么执行：
  - 设计会话数据结构（会话 ID、轮次、消息）
  - 实现本地历史持久化
  - 增加“继续上次会话”命令
- 产出什么：
  - 多轮对话能力
  - 会话历史文件
  - 10 轮对话样例与复盘

## 第 4 周：交付 MVP
- 任务内容：形成可演示版本
- 怎么执行：
  - 整合核心功能（总结、改写、提纲）
  - 提供统一入口（CLI 或 Web）
  - 走通一次端到端演示
- 产出什么：
  - 可演示 MVP
  - 演示脚本
  - 第一版迭代清单
