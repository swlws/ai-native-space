# memory

对话摘要

```text
Frontend messages[]
       → compressMessages()          ← 新增 memoryService
           ├─ 消息数 ≤ KEEP_RECENT: 直接透传
           └─ 消息数 > KEEP_RECENT:
                 older = messages[0 .. n-KEEP_RECENT]
                 recent = messages[n-KEEP_RECENT .. n]
                 summary = LLM(older)  ← 调用 chat()，非流式
                 返回 [system: summary, ...recent]
       → chatStream(compressedMessages)   ← 现有流程
```

memory 格式

```txt
  ┌──────────┬───────────────────────────────┬──────────────────────────────────────────────────┐
  │          │            改进前             │            改进后（参考 Claude Code）            │
  ├──────────┼───────────────────────────────┼──────────────────────────────────────────────────┤
  │ 存储结构 │ summary: string（自由文本块） │ memories: Memory[]（结构化分类）                 │
  ├──────────┼───────────────────────────────┼──────────────────────────────────────────────────┤
  │ 记忆类型 │ 无分类                        │ context / preference / decision / fact           │
  ├──────────┼───────────────────────────────┼──────────────────────────────────────────────────┤
  │ 每条记忆 │ 无索引                        │ description（一句话索引）+ content（详情）       │
  ├──────────┼───────────────────────────────┼──────────────────────────────────────────────────┤
  │ 压缩策略 │ 全量重生成摘要                │ 增量提取：仅处理溢出部分，LLM 负责与现有记忆合并 │
  ├──────────┼───────────────────────────────┼──────────────────────────────────────────────────┤
  │ LLM 注入 │ 一段平铺文字                  │ 按类型分节的结构化 system prompt                 │
  ├──────────┼───────────────────────────────┼──────────────────────────────────────────────────┤
  │ 循环依赖 │ session ↔ memory              │ 消除：两者均直接从 lib/llm 引入类型              │
  └──────────┴───────────────────────────────┴──────────────────────────────────────────────────┘
```

Claude Code 的 ReAct 设计

  Claude Code 的 ReAct 是一种协议层面的强类型实现，与论文版的"靠 prompt 约定格式"有本质区别。

  ---
  核心机制：以 stop_reason 驱动循环

  发请求（携带 tools 定义）
    └─ stop_reason = "tool_use"  → 执行工具 → 追加 tool_result → 继续请求
    └─ stop_reason = "end_turn"  → 退出循环

  Thought / Action / Observation 通过消息内容块隐式实现，没有显式文本标签：

  ┌─────────────┬──────────────────────────────────────────────────────┐
  │ ReAct 概念  │                   Claude Code 实现                   │
  ├─────────────┼──────────────────────────────────────────────────────┤
  │ Thought     │ {"type": "text"} 或 {"type": "thinking"} 块          │
  ├─────────────┼──────────────────────────────────────────────────────┤
  │ Action      │ {"type": "tool_use", "name": ..., "input": {...}} 块 │
  ├─────────────┼──────────────────────────────────────────────────────┤
  │ Observation │ {"type": "tool_result", "content": ...} 块           │
  └─────────────┴──────────────────────────────────────────────────────┘

  ---
  与论文版 ReAct 的关键差异

  ┌──────────────┬────────────────────┬─────────────────────────────────────────────────┐
  │     维度     │    论文版 ReAct    │                   Claude Code                   │
  ├──────────────┼────────────────────┼─────────────────────────────────────────────────┤
  │ Thought 标记 │ 显式 Thought: 文本 │ 隐式 text content block                         │
  ├──────────────┼────────────────────┼─────────────────────────────────────────────────┤
  │ 结构化       │ Prompt 约定        │ JSON Schema 强类型，strict: true 可消除无效调用 │
  ├──────────────┼────────────────────┼─────────────────────────────────────────────────┤
  │ 并行 Action  │ 不支持             │ 单轮多个 tool_use 块，可并发执行                │
  ├──────────────┼────────────────────┼─────────────────────────────────────────────────┤
  │ 错误恢复     │ Prompt 处理        │ is_error: true 标准字段，模型自动重试 2-3 次    │
  └──────────────┴────────────────────┴─────────────────────────────────────────────────┘

  ---
  对当前项目的参考意义

  我们当前的 react/index.ts 实现与 Claude Code 的思路基本一致，但有几点可以借鉴改进：

  1. 并行工具调用：Claude Code 单轮可执行多个工具并发返回，我们目前是串行的
  2. 错误透传：可在 tool_result 中加 is_error 字段，让模型自主决定是否重试
  3. 迭代上限语义：Claude Code 用 task_budget（token 预算）而非固定次数做软约束，比我们的 MAX_ITERATIONS = 8 更优雅
